### Conteúdo

O runbook documenta 3 passos sequenciados para promover queries históricas com `judge_score < 3` em tickets de curadoria:

#### Passo 1: Preview Count
```sql
SELECT COUNT(*), COUNT(DISTINCT query_id), COUNT(DISTINCT agent_id)
FROM agent_response_flags arf
WHERE judge_score < 3 
  AND arf.query_id NOT IN (SELECT ... FROM akp_curation_backlog)
  AND created_at < NOW() - INTERVAL '7 days'
```
Retorna: count de records low-judge, unique queries, agents afetados.

#### Passo 2: UPDATE No-op → Force Trigger
```sql
UPDATE agent_response_flags
SET judge_score = judge_score  -- no-op; força reativação do trigger
WHERE judge_score < 3 
  AND query_id NOT IN (...) 
  AND created_at < NOW() - INTERVAL '7 days'
```
Reactive `agent_response_flags_auto_flag` sem mudar dados (idempotente).

#### Passo 3: promote_gaps_to_backlog(3, 3.0, 'all_low_score')
```sql
SELECT promote_gaps_to_backlog(
  min_turns := 3,
  gap_threshold_pct := 3.0,
  judge_pattern := 'all_low_score'
)
```
Cria tickets em `akp_curation_backlog` com source_pattern v4.9.
